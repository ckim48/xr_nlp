About NLP Team

The objective of the NLP team was to develop a visual representation of a conversational interface for communication with virtual reality characters. This involved training the members to analyze and get valuable information from stored chat data by employing techniques such as keyword extraction and summarization.

Features & Purposes

The NLP Team has created a web-based dashboard with two core functionalities: Dialogue Analysis and Trend Analysis. These utilities use advanced natural language processing methods to shed light on conversations and datasets. The site achieves good analysis in areas such as summarization, sentiment, and keyword extraction through integration of models like GPT-3.5 and KeyBERT.

Page 1: Dialogue Analysis

Purpose

On Page 1, users can summarize input dialogue, analyze its sentiment, and extract keywords from it. The goals of the Dialogue Analysis page are to give users a full view of dialogues textually. Any dialogue can be inputted by a user, who will then receive a short summary, sentiment analysis, and keyword extraction.

Dialogue Summarization

  • Utilizes the GPT-3.5 model to summarize the input dialogue, providing a concise summary in 5 sentences.
  • Produces a summary in five sentences, capturing the essence of the dialogue.

Sentiment Analysis

  • Analyzes the sentiment of the input dialogue, providing three sentiment words and explanations.
  • Outputs three sentiment words and detailed explanations, helping users understand the emotional tone.

Keyword Extraction

  • Extracts the top 10 keywords from the dialogue using the KeyBERT model.
  • Provides keywords along with their relevance scores, highlighting key topics and themes.

Page 2: Trend Analysis

Purpose

Page 2, Users upload CSV data files which are used for emotion detection among other things within such information sets as well as showing trends relating these emotions or words over time periods specified by those datasets provided . The Trend Analysis page is more suited towards individuals working with big amounts of data while still requiring emotional mapping points that are relative enough to each other since this helps people recognize what could be happening when looking at large datasets without necessarily going into details concerning every single record contained in such set thus enabling identification even if not deep analysis because uploads involve many records rather than few ones only .

Data File Upload

  • Users can upload a CSV data file containing dialogue data.

Sentiment Analysis

  • Visualizes emotions extracted from the uploaded data.
  • Presents the data using various visual formats, such as word clouds, bar charts, and pie charts, making it easier to interpret.

Keyword Analysis

  • Visualizes the keywords extracted from the uploaded data.
  • Uses formats like word clouds, treemaps, and pie charts to display keyword distribution and importance.
